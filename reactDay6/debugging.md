# Day 06 - React Learning

## Topic Covered

### Debugging in React

Debugging is the process of finding and fixing errors (bugs) in an application.

---

## Why Debugging is Important

- Identify coding mistakes
- Improve application performance
- Understand application behavior
- Fix runtime errors

---

## Browser Developer Tools

Browser Developer Tools are built-in debugging tools available in modern browsers.

### Features

1. View HTML Structure
2. Inspect CSS Styles
3. Modify Styles in Real Time
4. View Console Messages
5. Execute JavaScript
6. Check Network Requests
7. Test Responsive Designs

---

## Console

Useful Methods:

console.log()

console.error()

console.warn()

Purpose:
- Display variable values
- Track program execution
- Identify issues

---

## React Developer Tools

React Developer Tools is a browser extension used for debugging React applications.

### Features

1. Inspect Components
2. View Props
3. View State
4. Track Component Hierarchy
5. Monitor Component Updates

---

## Benefits of React Developer Tools

- Easier debugging
- Better understanding of component structure
- Faster issue identification
- Improved development workflow

---

## Key Learnings

- Debugging process
- Browser Developer Tools usage
- React Developer Tools usage
- State and Props inspection
- Error identification techniques

---

## Practice Summary

Learned how to debug React applications using Browser Developer Tools and React Developer Tools.
